Development workflow

In the Tink development, we use the following branch name scheme.

  • feature/feature-name
  • bugfix/bugfix-name

Note that branchnames are written in all lower-case and dashes to avoid conflicts.

To be sure that we can add your contribution to the Tink framework, please follow the guidelines below.
The scheme for hotfixes is a little different, but these fixes will be resolved internally.

Be sure to follow the code guidelines mentioned on these pages if u want a greater chance for your pull request to be approved.

  1. Create a new branch (feature or bugfix) from the development branch.
  2. Commit and develop on the new branch because direct commits on the development branch are not allowed.
  3. Always merge the development branch in your branch before issuing a pull request.
  4. After your feature is tested locally, issue a pull request so we can merge it in the development branch.
