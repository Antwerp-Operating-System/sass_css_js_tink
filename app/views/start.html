What's included

Tink provides you with:

  • Basic pre-compiled styling of web components and HTML elements
  • Uniform use of typography and brand colors

What's coming?

Tink, why?

To have a good, future-ready responsive framework that fits the specific purpose of the apps we're building at Digipolis.
And to have a uniform way of visualising stuff and behaviours throughout various apps.

Tink, how?

Learning from experience

We set it up as a front-end framework with a lot of good coding practices, our UX principles, visual stying and with taking note of other. more generalistic platforms out there. That means for instance, for a lot of syntax, we piggy-back on the well-known bootstrap template language. So people can get up and running pretty quick without having to learn a complete different syntax.

Building on future experience

More components will be added in future versions, and also, modules that get built for one project, will be added to the framework library later. Hence creating a more powerful toolkit which will speed up development work.

Atomic approach

Tink is being built with the ideas of Atomic Design in mind, where each single HTML element symbolizes an atom, be it a form input, a button or a heading. An atom, a building block. Not really useful by themselves, but when we combine them with other atoms they become more useful. A form input together with a label, a helper text and a button can be seen as a molecule, which can serve as a newsletter signup block. These molecules serve as bigger building blocks or modules which can be easily replicated and together form an organism, a page layout, an interface.

Accessibility WAI-ARIA

Tink framework cares for accessibility, and tries as best as possible to comply with the WCAG 2.0 accessibility guidelines. That's why we focused on the inclusion of the necessary WAI-ARIA role elements, properties and states and relationships of elements. ARIA gives you the ability to augment the perceived meaning and semantics of your HTML. First of all, to improve the information provided to assistive technology like screen readers and second, to enhance useage in an accessible way.
